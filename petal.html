Petal — Agente de ventas
para floristería online

Petal es un agente conversacional completo diseñado para gestionar el ciclo de ventas de una floristería online ficticia, desde la consulta inicial hasta el checkout final.

User Stories

Arquitectura del sistema

El agente implementa una arquitectura multi-routine con tres capas principales:

ORQUESTADOR
Gestiona la intención del usuario y enruta a los playbooks especializados
CHECKOUT
Procesa pedidos con validación de stock, límites y direcciones
GESTIÓN DEUDA
Maneja clientes con pagos pendientes y define condiciones

Herramientas y backend

El sistema se conecta en tiempo real a un backend en Cloud Run que accede a Google Sheets como base de datos, permitiendo:

  • Consulta de inventario con disponibilidad actualizada
  • Validación de límites de crédito por perfil de cliente
  • Registro de pedidos y actualización de estado de pagos
  • Gestión de direcciones habituales y preferencias de entrega

Metodología aplicada

El proyecto sigue un proceso estructurado con User Stories y EPICs documentadas en Notion. Se implementó testing iterativo con perfiles de usuario reales y casos edge (stock insuficiente, deuda bloqueante, SLA excedido).

Prueba el agente en vivo

  • • "Quiero comprar flores"
